다산제약 홈페이지 구축 프로젝트 완료 일정 보고서
수행 기간: 2026. 06. 01 ~ 2026. 09. 30 (4개월)
작성일: 2026. 06. 02
보고서 작성자: IT 개발실
1. 프로젝트 개요
본 보고서는 다산제약의 디지털 브랜드 가치를 극대화하고, 글로벌 바이오 헬스케어 리더로서의 정체성을 확립하기 위한 공식 웹사이트 구축 프로젝트의 전체 주차별 완료 일정과 상세 계획안을 다룹니다.
본 웹사이트 개발은 컴포넌트 기반 아키텍처와 뛰어난 성능을 제공하는 Next.js 최신 프레임워크 및 Tailwind CSS v4를 활용하여 구축되고 있으며, PC와 모바일 기기 모두에 최적화된 반응형 웹 인터페이스를 탑재합니다.
2. 프로젝트 추진 방향 및 핵심 과제
• 브랜드 아이덴티티 통일: 홈페이지의 세련된 다크 및 청록색(Teal) 디자인을 모든 서브페이지 레이아웃에 확장 적용하여 고급스러운 비주얼 아이덴티티를 확립합니다.
• 다크 유리모핑 사이드바 메뉴: 상단 헤더와 자연스럽게 융합되는 다크 네이비 유리모핑 디자인을 서브 메뉴에 적용하여 프리미엄 사용자 경험(UX)을 구축합니다.
• 성능 최적화 및 보안: Next.js Turbopack 최적화 기술과 이미지 압축, 철저한 API 입력 검증으로 빠른 응답 속도와 완벽한 보안 환경을 확보합니다.

4. 월별 세부 개발 추진 계획
6월 - 기획 및 설계 (Planning &amp; Analysis Phase)
6월 한 달 동안은 전체 프로젝트의 뼈대를 놓는 기획 및 상세 설계 단계가 수행됩니다. 1~2주차에는 프로젝트의 요구사항 명세서(SRS)를 확정하고, 정보 구조(IA) 및 사이트맵을 도출해 대메뉴와 서브메뉴의 흐름을 확정합니다. 3~4주차에는 실제 개발의 밑그림이 될 화면 와이어프레임과 스토리보드를 작성하여 디자인과 프론트엔드 개발팀이 즉각 착수할 수 있도록 상세 설계 프로세스를 완료합니다.
7월 - 프론트엔드 및 화면 구현 (UI/UX Development Phase)
7월은 디자인 가이드와 컴포넌트를 설계하고 웹 화면을 개발하는 단계입니다. 1주차에는 브랜드 아이덴티티에 입각한 색상표, 버튼, 인풋 등 컴포넌트 마크업 구조를 설계하여 디자인 일관성을 정립합니다. 2~3주차에는 Next.js 기반 위에서 메인 비주얼 슬라이더 화면과 모던한 다크 서브페이지 레이아웃을 구현하고, 4주차에는 반응형 모바일 메뉴 및 영문 지원 다국어 서비스를 완료합니다.
8월 - 백엔드 개발 및 데이터 연동 (Backend &amp; DB Integration Phase)
8월에는 백엔드 API 서버를 구축하고 데이터베이스(MySQL)와 연동하는 작업을 집중 진행합니다. 1~2주차에는 보안과 트랜잭션을 고려한 DB 스키마 모델 모델링을 마치고, 1:1 고객 문의 접수 및 내역을 조회하는 핵심 백엔드 기능 개발을 완료합니다. 3~4주차에는 웹 관리자 페이지를 연동하여 모니터링 환경을 제공하고, 외부 API 및 도메인 보안 정책 설정을 완비합니다.
9월 - 테스트, 안정화 및 배포 (QA &amp; Launch Phase)
마지막 9월에는 완성된 홈페이지의 품질 보증(QA) 및 서버 배포 프로세스가 활성화됩니다. 1~2주차에는 다양한 브라우저와 모바일 기기에서의 UI 렌더링 최적화를 거친 뒤, 빌드 에러 모니터링 및 Next.js Turbopack 최적화 작업을 병행합니다. 3~4주차에는 실사용자 최종 검토(UAT) 피드백에 따른 긴급 보완 조치를 적용하고 운영 서버에 SSL 보안 인증 및 실서버 배포를 최종 오픈합니다.
5. 위험 관리 및 비상 계획
• 개발 일정 지연: Next.js 컴포넌트 재사용률을 최대화하여 설계 및 마크업 일정을 10% 이상 단축합니다.
• 브라우저 호환성: 퍼블리싱 단계부터 W3C 및 크로스 브라우징 가이드라인을 사전에 준수하여 크롬, 사파리, 엣지 간 편차를 최소화합니다.
• 보안 취약성: 사용자 입력값에 대한 철저한 SQL Injection 방지 및 API 파라미터 정규식 검증으로 데이터 보안을 최우선으로 확보합니다.
