# 중/소메뉴별 개별 SEO 설정 연동 내역 (2026-06-15)

본 문서는 다산제약 홈페이지의 세부 소메뉴(실제 주소)별 독립적인 SEO 설정 연동 작업에 대해 정리한 문서입니다. 본 연동을 통해 관리자 페이지에서 각각의 세부 주소별로 검색엔진 최적화 타이틀, 키워드, 설명을 완전히 분리하여 관리할 수 있게 되었습니다.

---

## 1. 개요 및 구현 전략

* **하이브리드(Fallback) 구조:** 
  1. 각 세부 페이지 로딩 시, 먼저 해당 페이지 고유의 SEO 설정(예: `seo/about/location`)을 데이터베이스에서 조회합니다.
  2. 만약 개별 입력된 데이터가 없다면, 상위 카테고리 기본 SEO 설정(예: `seo/about`)을 불러옵니다.
  3. 카테고리 설정마저 없다면 홈페이지 메인 SEO(`seo/main`)를 불러와 렌더링에 실패하지 않도록 다중 안전장치를 구성했습니다.
* **유연한 데이터 자동 등록 (Upsert):**
  * 데이터베이스에 기본 레코드가 미리 심겨 있지 않더라도, 관리자가 관리자 페이지에서 신규 소메뉴의 설정을 입력하고 '저장하기'를 누르면 `ON DUPLICATE KEY UPDATE` 구문을 통해 데이터베이스에 해당 레코드가 실시간으로 즉시 자동 추가되도록 구성했습니다.

```typescript
export async function generateMetadata({ params }: Params): Promise<Metadata> {
  const resolvedParams = await params;
  const slug = resolvedParams.slug || [];
  const pageKey = `about/${slug.join('/')}`;

  try {
    // 1. 개별 페이지 SEO 먼저 조회
    let results = await query('SELECT content FROM admin_contents WHERE page_key = ?', [`seo/${pageKey}`]);
    
    // 2. 없을 경우 대분류 SEO 상속 (Fallback)
    if (!results || results.length === 0 || !results[0].content) {
      results = await query('SELECT content FROM admin_contents WHERE page_key = ?', ['seo/about']);
    }
    
    // 3. 메타데이터 바인딩...
  } catch (e) { ... }
}
```
